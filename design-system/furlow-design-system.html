FURLOW

Design System v0.2

Visual language for FURLOW (Flexible User Rules for Live Online Workers). A YAML-driven Discord bot configuration framework named after the desert depot mechanic from Farscape who reverse-engineered wormhole tech from salvaged parts. Dark industrial. No polish. No bullshit. It works or it doesn't.

00 Logo & Identity

The FURLOW mark is a stack of three toggle switches -- two ON, one OFF. A dashed rail connects them on the left. It's a literal representation of what FURLOW does: feature switches for Discord bot configuration. The middle OFF toggle uses a dashed border ("furloughed") while the active ones are solid with a subtle orange fill.

Primary Mark

Mark Anatomy

Active Toggles

Top and bottom switches are ON. Solid 2px stroke border with orange fill at 20% opacity. Knob is a solid filled rectangle positioned at the right end of the track.

Furloughed Toggle

Middle switch is OFF. Dashed stroke border (4 3 pattern), no background fill. Knob at left end with 35% opacity. The dashed border is the core FURLOW visual motif: disabled/dormant state.

Connection Rail

Dashed vertical line on the left (3 4 pattern) connecting all three toggles. Small square indicator dots at each switch level. Active dots are full opacity, furloughed dot matches at 35%.

Proportions

120x120 viewBox. Toggles are 80x20 each with 10px vertical gap. Knobs are 28x16 (inset 2px from track edges). Rail sits at x=10 with 6x4 indicator dots. All geometry is rectangular, zero radius.

Below 32px: Drop the connection rail and indicator dots. Increase stroke widths to maintain readability. The three-bar silhouette is the minimum recognizable form.

Clearspace

Minimum clearspace around the mark equals the height of one toggle track (20 units in the 120-unit viewBox, or roughly 1/6th of the mark height). This keeps the logo breathing room without being precious about it.

Wordmark Lockups

Wordmark uses Oswald 700, uppercase, with 5px letter-spacing. "FUR" in heading white, "LOW" in accent orange. The mark sits to the left of the text, vertically centered. Minimum gap between mark and text is 16px at standard sizes.

Usage Rules

Do Don't
Use the mark on dark backgrounds (primary context) Place on busy images or patterned backgrounds
Use --accent-dark (#993d1e) variant on light surfaces Use the bright orange mark on white without swapping
Maintain the ON/OFF/ON toggle order Rearrange toggle states or remove any switch
Scale uniformly, keep all elements proportional Stretch, skew, rotate, or add effects to the mark
Drop the rail below 32px for readability Add the rail back at small sizes where it becomes noise
Use the wordmark lockup for headers and splash screens Stack the wordmark below the mark (always horizontal)
Keep dashed borders on the OFF toggle in all contexts Make all three toggles identical (removes the meaning)

06 Code & Syntax

FURLOW configs are YAML. The syntax theme is tuned specifically for YAML readability on dark backgrounds. Every color has a job. If two things look the same, they mean the same thing.

Then /verify."

Code Block Anatomy

Code blocks use --bg-code (#0e0e0e), slightly darker than base. A 3px left border in --accent-dim marks the block edge. An optional label sits top-right in --bg-panel. Line height is 1.8 for readability. Horizontal overflow scrolls, never wraps.

Inline Code

Use action: reply to send messages. Set ephemeral: true for private responses. Expression syntax: ${user.id}.

11 Motion

Animations are fast and purposeful. Nothing bounces. Nothing floats. Things appear, shift, or glow. Default duration: 150ms for micro-interactions, 300ms for transitions, 0ms for unnecessary animation. Easing: ease-out for entrances, ease-in for exits. Hover on these to preview.

12 Design Principles
Built from Scrap
Furlow the character builds impossible tech from junkyard parts. The design system reflects that same resourcefulness. No third-party component libraries. No CSS frameworks. Two fonts, one accent color, raw CSS. That is the entire toolkit.
Function Over Form
If an element does not communicate something, remove it. Decoration is noise. Rounded corners are decoration. Gradient backgrounds are decoration. Drop shadows on cards are decoration. Borders and background shades are information.
Dark by Default
There is no light theme. There never will be. The interface is a dark workshop lit by the glow of a terminal and the orange spark of a welding torch. That metaphor drives every color decision.
YAML is the UI
FURLOW is a YAML config tool. The visual design exists to make YAML legible and navigable. The syntax highlighting, the code block anatomy, the sidebar hierarchy: they all serve the config file. If the design fights the YAML, the design is wrong.
No Polish
Dashed borders over solid where possible. Noise grain over clean gradients. Monospace over proportional. Uppercase over title case. The aesthetic says "this was built in a garage" not "this was approved by a design committee."
Ecosystem Kinship
FURLOW sits alongside CLASP, LATCH, SLAB, and LumenCanvas. They share DNA (monospace type, dark backgrounds, maker energy) but each has its own accent and personality. FURLOW's orange separates it from CLASP's teal and SLAB's neutral palette while keeping the family resemblance unmistakable.

Do / Don't

Do Don't
Use Oswald for headings, always uppercase Use Oswald for body text or mixed case
Use dashed borders for separators and placeholders Use rounded corners on anything
Use the noise overlay on every page Use background images or complex textures
Use orange accent sparingly for emphasis Saturate the page with accent color
Use semantic colors only for their purpose Use green for decoration or blue for headings
Animate with 150ms transitions Bounce, spring, or overshoot anything
Use Font Awesome solid icons at 11-14px Use emoji, line icons, or custom illustrations
Keep code blocks left-aligned with accent bar Center code or use pastel syntax themes
